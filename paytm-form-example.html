Paytm Payment Gateway - Auto-Submitting Form Example

⚠️ CRITICAL: The Paytm URL https://securegw.paytm.in/theia/processTransaction MUST NEVER be opened directly in a browser. It only accepts POST requests from HTML forms.
✅ Correct Flow:
  1. Backend generates Paytm parameters and checksum
  2. Frontend receives those parameters
  3. Frontend creates a hidden HTML form (method=POST)
  4. Frontend auto-submits the form to Paytm
  5. User completes payment on Paytm's page

Example: Auto-Submitting Form

Click the button below to simulate the payment flow:

JavaScript Code:

function redirectToPaytm(paytmParams) {
  // Remove any existing form
  const existingForm = document.getElementById('paytm-payment-form');
  if (existingForm) existingForm.remove();

  // Create hidden HTML form
  const form = document.createElement('form');
  form.method = 'POST'; // MUST be POST
  form.action = 'https://securegw.paytm.in/theia/processTransaction';
  form.target = '_self';
  form.style.display = 'none'; // Hide the form
  form.id = 'paytm-payment-form';

  // Add all parameters as hidden inputs
  Object.keys(paytmParams).forEach((key) => {
    const input = document.createElement('input');
    input.type = 'hidden';
    input.name = key;
    input.value = paytmParams[key];
    form.appendChild(input);
  });

  // Append to body and auto-submit
  document.body.appendChild(form);
  form.submit(); // This sends POST request to Paytm
}

Key Points:

  • Form method MUST be POST - GET requests are rejected
  • Form must be hidden - Use style.display = 'none'
  • All parameters required - MID, ORDER_ID, TXN_AMOUNT, CHECKSUMHASH, etc.
  • Auto-submit programmatically - Not via user click on link
  • Never open URL directly - Always use form POST submission
